# 英雄控制

> **状态**：草稿
> **归属区域**：Client + Shared
> **最后更新**：2026-03-24

---

## 系统概述

英雄控制系统负责玩家对英雄的基础操作体验，包括移动、朝向、目标选取、技能施放输入和本地反馈。它决定这款 MOBA “玩起来像不像样”。

## 玩家体验目标

玩家应该感到移动响应及时、朝向清晰、技能释放不拖泥带水，尤其在手机端也能完成基本走位、追击、撤退和技能命中。

## 核心职责

- 接收并整理玩家输入
- 控制英雄移动意图与面向方向
- 提供目标锁定、目标切换或最近目标辅助
- 将技能释放意图传递给服务端战斗系统
- 在本地提供范围提示、指示器和按钮反馈

## 输入

- 键盘、鼠标、手柄或触屏输入
- 当前英雄状态，如是否眩晕、沉默、死亡
- 当前可选目标信息
- 对局状态

## 输出

- 移动意图
- 朝向或瞄准意图
- 普攻请求
- 技能施放请求
- 本地范围提示和技能按钮冷却表现

## 规则与状态变化

1. 玩家只有在 `正式对局` 状态且英雄存活时才可执行有效操作。
2. 移动使用第三人称相对镜头方向驱动，保证 PC 和 Mobile 输入表现尽量统一。
3. 普攻优先攻击当前锁定目标；若无锁定目标，则攻击有效范围内最近的敌对目标。
4. 技能分为指向型、范围型和锁定型三类，统一通过施放请求发送到服务端。
5. 当英雄受到硬控时，部分或全部操作被禁止，但 UI 仍需明确显示原因。

## 服务端权威与复制策略

- 最终裁决方：Mixed
- 远程通信需求：
  - `BasicAttackRequest`
  - `AbilityCastRequest`
  - `TargetSelectionRequest`
- 反作弊校验点：
  - 服务端校验技能是否可用、目标是否合法、施放距离是否合理
  - 对异常移动速度、非法位移和超范围请求进行拦截
- 需要复制给客户端的状态：
  - 控制禁用状态
  - 当前目标
  - 技能冷却结果
  - 朝向修正或关键位移结果

## 数据模型

| 字段 | 类型 | 持有方 | 说明 |
| --- | --- | --- | --- |
| MoveVector | Vector3 | Client | 当前移动方向输入 |
| AimDirection | Vector3 | Client | 当前瞄准方向 |
| CurrentTargetId | string | Client + Shared | 当前锁定目标 |
| ControlState | string | Shared | Normal、Stunned、Silenced、Dead 等 |
| AbilityInputBuffer | table | Client | 短时间输入缓存，便于处理连按 |

## 玩家交互流程

1. 玩家拖动摇杆或按键移动英雄
2. 系统根据镜头方向计算移动向量
3. 玩家点击普攻或技能按钮
4. 本地先显示施放指示和按钮反馈
5. 服务端验证后返回真正生效的技能或攻击结果

## 边界情况

- 玩家被控制时仍持续按技能
- 手机端误触导致目标丢失
- 锁定目标死亡或超出范围
- 高延迟下本地已经放技能，但服务端判定失败

## 可调参数

| 参数 | 默认值 | 安全范围 | 用途 |
| --- | --- | --- | --- |
| 目标吸附半径 | 10 | 6 到 16 | 帮助玩家更稳定选中目标 |
| 输入缓冲时长 | 0.15 秒 | 0 到 0.25 秒 | 避免手感过硬 |
| 技能指示器显示时长 | 按住期间 | 即时到持续显示 | 匹配不同技能类型 |
| 移动转向平滑度 | 中 | 低到高 | 调整操作手感 |

## 监控与审核注意点

- 建议监控指标：技能施放失败率、目标选取切换频率、移动端误触率
- 容易被滥用的点：伪造目标、超距离施法、异常位移

## 验收标准

- [ ] 玩家可以稳定完成移动、普攻和 3 个技能的基础输入
- [ ] 移动端玩家可以在不接外设的情况下完成基本战斗操作
- [ ] 被控制、死亡、技能冷却等状态能够明确反馈且不可绕过
